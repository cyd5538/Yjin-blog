# 시간 복잡도란 ?
**시간 복잡도는 특정 알고리즘이 얼마나 빠르게 수행이되는지 표현하기 위해 사용된다.**
즉, 시간복잡도는 쉽게 말해서 알고리즘의 실행 시간을 말한다.
## 시간 복잡도의 종류
* 최상의 경우 : 오메가 표기법 (Big-Ω Notation)
* 평균의 경우 : 세타 표기법 (Big-θ Notation)
* 최악의 경우 : 빅오 표기법 (Big-O Notation)

최악의 경우에도 효율적으로 알고리즘이 동작해야하기 때문에 <u>빅오 표기법을 주로 사용한다.⭐</u>

# Big O Notation ?
문자열을 거꾸로 출력하는 방법에만 해도 10가지가 넘는다. 그 중에서 어떤 것이 제일 좋은 방법일까 ?
이 중 어떤 방법이 더 효율적인지 **빅오 표기법**을 사용하여 평가해볼 수 있다. 빅오표기법은 수학적인 표기로 나타낸다.<br></br>

## O(n) - Linear Time
입력 크기에 비례하여 실행 시간이 증가한다.
```js
for (let i = 0; i < n; i++) {
    // 실행 코드
}
```

## O(log n) - Logarithmic Time
입력 크기에 대해 로그 함수의 형태로 실행 시간이 증가한다.
```js
for (let i = 1; i < n; i *= 2) {
    // 실행 코드
}
```

## O(n log n) - Linearithmic Time
입력 크기에 대해 선형적으로 증가하면서 로그 함수의 형태로 실행 시간이 증가한다.
```js
for (let i = 0; i < n; i++) {
    for (let j = 1; j < n; j *= 2) {
        // 실행 코드
    }
}
```

## O(n^2) - Quadratic Time
입력 크기의 제곱에 비례하여 실행 시간이 증가
```js
for (let i = 0; i < n; i++) {
    for (let j = 0; j < n; j++) {
        // 실행 코드
    }
}
```
