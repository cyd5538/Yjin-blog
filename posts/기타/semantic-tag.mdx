# 시맨틱 태그(Semantic Tag)
시맨틱 태그란 의미가 있는 태그란 뜻이다.  
`div`나 `span`과 같은 태그만으로 이루어진 페이지는 어떤 내용인지 전혀 유추할 수가 없는 반면에 
시맨틱 태그를 사용하면 태그의 이름만 보고도 역할이나 위치를 알 수 있다.

# 시맨틱 태그를 사용해야 하는 이유
* 검색엔진 최적화 SEO(Search Engine Optimization)에 유리하다.
* HTML 문서의 가독성이 향상된다.
* 웹 접근성이 향상된다. 시각 장애인들에게 스크린 리더기를 제공한다.

# 시맨틱 태그 종류

  <img src="/img/etc/semantic.jpg" alt="시맨틱 태그" width="800"/>
</p>
* `header`: 문서 전체 또는 섹션의 헤더를 나타내기 위해 사용된다. 일반적으로 웹사이트의 로고, 이름, 소개글, 그리고 메인 메뉴 등을 포함한다.
* `nav`: 네비게이션 또는 탐색을 위한 링크의 집합을 나타낸다. 보통 웹 페이지의 메뉴나 목차를 가르킨다.
* `main`: 문서의 중심 주제, 주요 내용 또는 응용 프로그램의 중심 기능과 직접 관련된 콘텐츠를 감싸기 위해 사용된다. 한 페이지에는 하나의 `main` 태그만 사용할 수 있다.
* `section`: 구체적인 시맨틱 태그가 없는 문서의 독립적인 영역을 나타낸다.
* `article`: 독립된 컨텐츠를 나타낸다. 기사, 블로그 포스트, 게시물 등과 같이 독립적으로 사용될 수 있는 콘텐츠 등이 있다.
* `aside`: 본문 외에 부수적인 내용을 포함하는 데 사용된다. 보통 광고나 사이드바를 표시할 때 사용한다.
* `footer`: 문서의 꼬리말이나 바닥글을 나타낸다. 주로 회사 소개, 약관, 저작권 등에 정보 등을 표시한다.
