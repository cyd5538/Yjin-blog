# 모듈
프로그래밍에서 `모듈`이라는 용어는 일반적으로 특정 기능이나 역할을 수행하는 작은 코드 단위를 의미한다.  
자바스크립트에서는 아래와 같은 코드를 모듈이라고 한다.

```js
// math.js
export function add(a, b) {
  return a + b;
}

export function subtract(a, b) {
  return a - b;
}

export { add, substract }
```
* 두 숫자의 합을 구하는 add함수
* 두 숫자의 차이를 구하는 substract함수

성격이 비슷한 기능들을 하나의 파일로 관리하면 모듈이 된다.  

> <u>그럼 웹팩에서의 모듈은 뭘까</u> ?  
웹팩(Webpack)에서의 모듈은 자바스크립트 파일 뿐만 아니라 다양한 유형의 파일( HTML, CSS, Javascript, Images, Font)들을 하나로 묶어 번들로 생성하는 개념이다.

# 웹팩
## 왜 웹팩이 등장했는가?
webpack이 등장한 이유를 알기 위해서는 번들러가 사용되기 전에 어떻게 JavaScript를 사용했는지를 알아야 한다.  브라우저에서 JavaScript가 동작하는 두 가지 방법이 있다.  
1. 각 기능이 있는 스크립트를 추가합니다. 이 방식은 너무 많은 스크립트로 인해 네트워크의 병목을 유발하는 원인이 될 수 있어서 확장이 어렵다.  
2. 모든 프로젝트의 거대한 `.js` 파일을 만들어 사용하는 것이다. 이것은 유효범위와 크기, 가독성, 유지보수에 문제를 발생시킬 수 있는다.

웹팩은 이러한 문제를 해결하고자 모듈 번들링 도구로 등장했다.

## 웹팩으로 해결하려는 문제?
**1. 자바스크립트 변수 유효 범위 문제**  
웹팩은 ES6의 모듈 문법을 활용하여 변수의 유효 범위 문제를 해결한다. 각 모듈은 독립적인 스코프를 가지기 때문에 전역 스코프의 변수 충돌을 방지하고 모듈 간의 의존성을 명시적으로 관리할 수 있다.

**2. 브라우저별 HTTP 요청 숫자의 제약**  
웹팩은 여러 개의 파일을 번들로 묶어서 단일 파일로 생성함으로써 브라우저에서 처리해야 하는 HTTP 요청 숫자를 줄이는 데 도움을 준다. 이를 통해 네트워크 병목 현상을 완화하고 초기 로딩 속도를 향상시킬 수 있다.
