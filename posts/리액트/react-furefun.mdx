# 순수함수란?
* 동일한 입력이 주어졌을 때 동일한 출력을 해야한다.
* 순수 함수는 외부 상태에 영향을 주지 않으며, 부수효과(Side Effect)가 없어야 한다.  

```jsx

function Number({num}) {
  return <h2>숫자는 {num}입니다.</h2>;
}

export default function App() {
  return (
    <>
      <Number num={2}/>
      <Number num={3}/>
      <Number num={4}/>
    </>
  );
}

```

# Side Effect
React 컴포넌트의 주요 목적은 상태(state)와 속성(props)을 관리하고 UI를 렌더링하는 것인데 이 외에 작업은
부수효과(Side Effect)라고 보면 된다. 대표적으로는 아래의 예시들이 있다.
* API 호출
* 로컬 스토리지 이용
* 화면 업데이트
* DOM 조작
* setTimeout, setInterval 등의 함수

React에서 사이드 이펙트는 보통 이벤트 핸들러에 속한다. 이벤트 핸들러는 사용자가 어떤 동작을 할 때 발생하는 이벤트다.
이벤트 핸들러는 렌더링 중에는 실행되지 않기 때문에 순수할 필요가 없다.

사이드 이펙트에 대한 마지막 수단으로 useEffect를 사용할 수 있다. 그러나 이는 최후의 수단으로 사용해야 한다.

# 요약
* 컴포넌트는 순수해야 한다.
* 순수 함수는 예측 가능하고 안정적이다.
* 렌더링 단계에서 변경 사항을 피하고, 사이드 이펙트는 `useEffect` 내에서 다루어야 한다.
